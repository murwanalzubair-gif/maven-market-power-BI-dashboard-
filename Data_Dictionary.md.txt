# Data Dictionary

## Fact Tables

### Fact_Transactions
Sales transaction line items (1997 + 1998 appended).

| Column | Type | Description |
|---|---|---|
| transaction_date | Date | Date the sale occurred |
| stock_date | Date | Date the product was stocked |
| product_id | Integer | Foreign key → Dim_Products |
| customer_id | Integer | Foreign key → Dim_Customers |
| store_id | Integer | Foreign key → Dim_Stores |
| quantity | Integer | Units sold in this line item |

Row count: ~269,720 (1997: 86,837 + 1998: 182,883)

---

### Fact_Returns
Product return line items (1997 + 1998).

| Column | Type | Description |
|---|---|---|
| return_date | Date | Date the return occurred |
| product_id | Integer | Foreign key → Dim_Products |
| store_id | Integer | Foreign key → Dim_Stores |
| quantity | Integer | Units returned in this line item |

Row count: 7,087

Note: Fact_Returns is a second Fact table (not a Dimension) — it holds transactional/event data, same as Fact_Transactions.

---

## Dimension Tables

### Dim_Products
| Column | Type | Description |
|---|---|---|
| product_id | Integer | Primary key |
| product_brand | String | Brand name |

| product_sku | Integer | Stock keeping unit |

## Relationships (Star Schema)
- Fact_Transactions[product_id] → Dim_Products[product_id]
- Fact_Transactions[customer_id] → Dim_Customers[customer_id]
- Fact_Transactions[store_id] → Dim_Stores[store_id]
- Fact_Transactions[transaction_date] → Dim_Calendar[date]
- Fact_Returns[product_id] → Dim_Products[product_id]
- Fact_Returns[store_id] → Dim_Stores[store_id]
- Fact_Returns[return_date] → Dim_Calendar[date]
- Dim_Stores[region_id] → Dim_Regions[region_id]
